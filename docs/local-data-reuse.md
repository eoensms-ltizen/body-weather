# Local Atlas Reuse

Body Weather는 계정이나 서버 업로드 없이 새로고침 뒤 분석 결과를 다시 사용할 수 있도록 선택형 기기 저장을 제공한다. 저장 위치는 현재 사이트 Origin의 브라우저 IndexedDB이며, 한 번에 최신 Atlas 한 세트만 유지한다.

## 사용자 흐름

1. ZIP을 선택하고 `분석 결과를 이 기기에 기억`을 직접 켠다.
2. 가져오기가 완료되면 정규화된 `ImportSummary`를 IndexedDB에 저장한다.
3. 다음 방문의 첫 화면에서 날짜 범위, 활동·경로 수, 저장 시각만 먼저 읽어 `이전 Atlas 이어보기`를 표시한다.
4. 사용자가 `이어보기`를 눌렀을 때만 전체 분석 결과를 읽고 Atlas Worker를 다시 실행한다.
5. 첫 화면 또는 `Data & Privacy`에서 저장 데이터를 삭제하거나 현재 Atlas로 교체한다.

## 저장 범위와 비저장 범위

저장:

- 정규화된 활동 필드와 원본 GPS route point
- Garmin 웰니스 결과
- Capability, 데이터 품질 진단, 소스 매칭 결과
- 복원 카드에 필요한 비식별 집계 메타데이터

저장하지 않음:

- Strava/Garmin ZIP Blob 자체
- 사진과 영상
- 운영체제의 ZIP 절대경로
- 현재 지도 위치, 선택 활동, 필터, 숨김 목록, Premiere 재생 위치
- 서버, D1, R2, 분석 서비스의 사용자 데이터

## 개인정보·보안 경계

- IndexedDB 데이터는 전송 중인 데이터가 아니라 이 브라우저 프로필의 디스크에 남는 데이터이며 앱 자체 암호화를 적용하지 않는다.
- 같은 기기·브라우저 프로필에 접근할 수 있는 사람과 같은 Origin에서 실행되는 앱 코드는 저장 데이터에 접근할 수 있다. 공용 PC에서는 사용하지 않는다.
- 지도 표시와 Poster에서는 복원 뒤에도 기존 민감 위치 마스킹을 다시 계산한다.
- 브라우저 사이트 데이터 삭제 또는 앱의 `저장 데이터 삭제`로 제거한다. `새 ZIP`은 현재 화면만 초기화하고 저장 데이터는 지우지 않는다.
- 저장 공간 부족, 사생활 보호 모드, IndexedDB 미지원에서는 현재 세션만 유지하고 원인을 표시한다.

## 데이터 계약

- 데이터베이스: `body-weather-local-atlas`
- Object stores: `summaries`, `metadata`
- 단일 key: `latest-atlas`
- schema version: `1`
- metadata 조회는 전체 GPS/웰니스 레코드를 읽지 않는다.
- 두 Object store는 하나의 read-write transaction으로 저장·삭제해 부분 기록을 방지한다.

## 검증 게이트

- [x] 메타데이터에 원본 파일명과 `sourceFile`을 포함하지 않는다.
- [x] 경로가 있는 활동·point 수와 소스 조합을 결정적으로 집계한다.
- [x] IndexedDB가 없는 환경에서는 예외 대신 저장 없음으로 저하된다.
- [x] TypeScript, ESLint, 전체 단위 테스트, 프로덕션 빌드.
- [ ] 실제 대용량 ZIP 결과에서 브라우저 quota와 저장·복원 시간 측정.
- [x] 실제 Chrome 자동 흐름에서 저장 → 새로고침 → 이어보기 → 삭제 확인.
- [ ] 배포 URL에서 실제 개인 데이터로 저장 용량과 복원 시간 최종 확인.
